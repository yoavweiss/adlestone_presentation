Image compression 101

Image formats

GIF

JPEG

God for real life photos. 8x8 blocks. Been around since early 90s

No alpha channel

Lossy compression

Converts color to frequencies, gets rids of high frequencies, entropy coding for the rest

PNG

Multiple variants

color types. The format is based on gzip, so it compresses continous changes better. (so CG images)

Lossless

PNG8

PNG 8 - includes up to 256 colors. Can have a full alpha channel Define a color palette, and then each pixel references a color number.

PNG24

PNG 24 - Each pixel can be a separate color (defined up to 24 bits precision + 8 bit alpha).

Alpha Channel

Unlike JPEG, the format supports a full alpha channel!!!

SVG

Scalable Vector Graphics. An ideal solution for respimg!!!

CG, not photos

It is great for computer generated graphics, but not so for real-life photos (even though it can embed them).

Markup based

So human readable/writable. OTOH, needs to be minified/gzipped.

Compare size vs. PNG

often, for small graphics, PNG can be smaller than the SVG equiv.

Not pixel perfect

But there's an SVG hinting spec being worked on

WebP

Google introduced it a few years back. Based on VP8 video.

Both lossy and lossless

Supports animation, alpha

Funny for a ex video format

Novelties

It offers better compression because of predictive coding, variable size blocks, and applying different compression levels to different parts of the image.

Poor cross-browser adoption

Mozilla are not convinced that the format is significantly better. Working on improving JPEG encoders.

JPEG-XR

Microsoft's attempt to improve image formats. Better at HD images, but less so for the Web

JPEG-XT

A backwards compatible attempt to add features to JPEG

Alpha channel, lossless

Combined with better encoders, might be good enough

Compression methods

Lossless compression

Gets rid of meta data, better lossless compression even on the lossy formats

Lossy compression

Drops quality in favor of smaller file size. Can be applied to lossless formats as well.
